# Custom

This page describes the configuration that is required to communicate with a custom API service.

## How To

The quickest way to connect to an existing service is to use the [`requestInterceptor`](#RequestInterceptor) and [`responseInterceptor`](#ResponseInterceptor) properties
(in [`ServiceCallConfig`](HERE)) which will allow you to modify the outgoing requests for your target API and alter the responses to allow the component to interpret them correctly.

By default, the **outgoing** request bodies will contain all of the messages within the chat (to allow you to interpret their context as needed). This behaviour can be altered within
the [`custom`](#custom) object by the use of properties such as [`maxMessages`](HERE) and [`totalMessagesMaxCharLength`](HERE).
The outgoing body can also contain additional properties by adding them to the [`custom`](#custom) object.

- When sending text based messages only, the request body will have the following JSON type: <br />
  {[`messages: MessageContent[]`](../messages/#MessageContent)} <br />

- When sending messages that contains files, the request body is going to be encapsulated inside a [FormData](https://developer.mozilla.org/en-US/docs/Web/API/FormData) type.

The **response** message body needs to have the following JSON type: <br />
&nbsp;&nbsp;&nbsp;&nbsp {[`result?: Result[]`](#Result), `error?: string`} <br />

Please see [`EXAMPLES`](https://github.com/OvidijusParsiunas/ai-assistant/tree/main/other-packages/servers) for setting up your own server.

<h2>Component setup</h2>

### `custom` {#custom}

- Type: [`ServiceCallConfig`](HERE) & [`MessageLimits`](#MessageLimits) & {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`images?: boolean | FilesServiceConfig`](#FilesServiceConfig), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`audio?: boolean | FilesServiceConfig`](#FilesServiceConfig), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`mixedFiles?: boolean | FilesServiceConfig`](#FilesServiceConfig), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`camera?: boolean | CameraFilesServiceConfig`](#CameraFilesServiceConfig), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`microphoneAudio?: boolean | MicrophoneFilesServiceConfig`](#MicrophoneFilesServiceConfig), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`stream?: boolean`](#stream), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`displayServiceErrorMessages?: boolean`](#displayServiceErrorMessages), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`demo?: boolean | Demo`](#Demo), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `string?: any` <br />
  }

Object used to configure AiAssistant to communicate with a custom API. It is a combination of the [`ServiceCallConfig`](HERE), [`MessageLimits`](#MessageLimits)
and other custom properties. <br /> Whilst everything is marked as optional, either a [`request`](HERE) or `demo` property needs to be set. `key` is not required. <br />
This object can also be populated with your own key (string) values (any) which will be added to the outgoing messages' request bodies. <br />

#### Example

import ActiveTable from '@site/src/components/table/activeTableBrowser';
import AiAssistantBrowser from '@site/src/components/table/aiAssistantBrowser';
import ComponentContainer from '@site/src/components/table/componentContainer';
import LineBreak from '@site/src/components/markdown/lineBreak';
import BrowserOnly from '@docusaurus/BrowserOnly';
import TabItem from '@theme/TabItem';
import Tabs from '@theme/Tabs';

<BrowserOnly>{() => require('@site/src/components/nav/autoNavToggle').readdAutoNavShadowToggle()}</BrowserOnly>

<ComponentContainer>
  <AiAssistantBrowser
    introMessage={`Hi, I am your assistant. How may I help you?`}
    service={{
      custom: {request: {url: 'https://customapi.com'}, images: {button: {position: 'outside-right'}}},
    }}
  ></AiAssistantBrowser>
</ComponentContainer>

## Types

Property types for the [`custom`](HERE) object:

### `ServiceCallConfig` {#ServiceCallConfig}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `key?: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`request?: RequestSettings`](#RequestSettings), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`requestInterceptor?: RequestInterceptor`](#RequestInterceptor), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`responseInterceptor?: ResponseInterceptor`](#ResponseInterceptor) <br />
  }

Configuration used to call a service and intercept the data transferred between it and the component.
If you are using one of the preconfigured services - you can add the `key` property to skip the `insert key view`
and immediately display the chat. If you want this key to be verified, set [`validateKeyProperty`](HERE) to _true_.

<LineBreak></LineBreak>

### `CameraFilesServiceConfig` {#CameraFilesServiceConfig}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`request?: RequestSettings`](#RequestSettings), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`files?: CameraFiles`](#CameraFiles), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`button?: Button`](#Button), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`modalContainerStyle?: CustomStyle`](HERE), <br />
  }

Configuration for capturing images using a web camera and handling the resultant files. <br />
`request` is used to override the default request settings when sending image files. <br />
`files` is used to configure the files that are captured by the camera. <br />
`button` is used to customize the button element used for the camera. <br />
`modalContainerStyle` is used to customize the modal used to display the camera. <br />

<ComponentContainer>
  <AiAssistantBrowser
    service={{
      custom: {
        demo: true,
        camera: {
          request: {url: 'https://customapi.com/image-from-camera'},
          files: {format: 'png'},
          button: {position: 'outside-left'},
          modalContainerStyle: {borderRadius: '5px'},
        },
      },
    }}
  ></AiAssistantBrowser>
</ComponentContainer>

<LineBreak></LineBreak>

### `CameraFiles` {#CameraFiles}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `format?:` `"png"` | `"jpeg"`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `dimensions?:` {`width?: number`, `height?: number`} <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `maxNumberOfFiles?: number`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `acceptedFormats?: string`, <br />
  }

- Default: _{<br />
  &nbsp;&nbsp;&nbsp;&nbsp; format: "png", <br />
  &nbsp;&nbsp;&nbsp;&nbsp; maxNumberOfFiles: ∞, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; acceptedFormats: "image/\*" <br />
  }_

Configuration for the captured files by the camera. <br />
`format` is the resultant format of the image file that is produced by capturing a photograph. <br />
`dimensions` are used to set the pixel area of the photograph. Please note that this is largely controlled by the [`mediaDevice`](https://developer.mozilla.org/en-US/docs/Web/API/MediaDevices/getUserMedia)
of the user's browser, so if the dimensions you have provided skew the photograph in a particular way they can be overriden.<br />
`maxNumberOfFiles` is the limit to the number of images that can be captured for one message. <br />
`acceptedFormats` is used to limit the allowed files that can be dropped via the [`dragAndDrop`](HERE) functionality as this feature will enable it automatically.
It uses the same syntax as the [accept](https://developer.mozilla.org/en-US/docs/Web/HTML/Attributes/accept) attributes
in [input elements](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input/file). If the configuration in [`images`](HERE) also uses
the `acceptedFormats` property, it will override this one.<br />

<ComponentContainer>
  <AiAssistantBrowser
    service={{
      custom: {
        demo: true,
        camera: {
          files: {format: 'png', dimensions: {width: 450, height: 600}, maxNumberOfFiles: 2, acceptedFormats: '.jpg,.png'},
        },
      },
    }}
  ></AiAssistantBrowser>
</ComponentContainer>

<LineBreak></LineBreak>

### `AudioRecordingFiles` {#AudioRecordingFiles}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`format?: AudioFormat`](HERE), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `maxDurationSeconds?: number`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `maxNumberOfFiles?: number`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `acceptedFormats?: string` <br />
  }

- Default: _{<br />
  &nbsp;&nbsp;&nbsp;&nbsp; format: "mp3", <br />
  &nbsp;&nbsp;&nbsp;&nbsp; maxDurationSeconds: 5999, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; maxNumberOfFiles: ∞, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; acceptedFormats: "audio/\*" <br />
  }_

Configuration for the captured files by the camera. <br />
`format` is the resultant format of the audio file. <br />
`maxDurationSeconds` is the maximum length of time one file can be recorded.<br />
`maxNumberOfFiles` is the limit to the number of audio files that can be recorded for one message. <br />
`acceptedFormats` is used to limit the allowed files that can be dropped via the [`dragAndDrop`](HERE) functionality as this feature will enable it automatically.
It uses the same syntax as the [accept](https://developer.mozilla.org/en-US/docs/Web/HTML/Attributes/accept) attributes
in [input elements](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input/file). If the configuration in [`images`](HERE) also uses
the `acceptedFormats` property, it will override this one.<br />

<ComponentContainer>
  <AiAssistantBrowser
    service={{
      custom: {
        demo: true,
        microphoneAudio: {
          files: {format: 'mp3', maxDurationSeconds: 10, maxNumberOfFiles: 2, acceptedFormats: '.mp3,.wav'},
        },
      },
    }}
  ></AiAssistantBrowser>
</ComponentContainer>

### `FilesServiceConfig` {#FilesServiceConfig}

- Type: {[`request?: RequestSettings`](HERE), [`FilesUploader`](HERE)}

`request` object is used to override the default service request settings when sending a file. <br />
`FilesUploader` is used to configure what files can be sent to the target service and change the styling of elements that facilitate this. <br />

<ComponentContainer>

### `FileAttachments` {#FileAttachments}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `maxNumberOfFiles?: number`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `acceptedFormats?: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`infoModal?: InfoModal`](#InfoModal) <br />
  }

- Default: _{<br />
  &nbsp;&nbsp;&nbsp;&nbsp; maxNumberOfFiles: for [`custom`](HERE): ∞, for existing services: varies, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; acceptedFormats: for images: ".png,.jpg", for audio: ".4a,.mp3,.webm,.mp4,.mpga,.wav,.mpeg,.m4a" <br />
  }_

Used to control what files the component can accept. <br />
`maxNumberOfFiles` controls the maximum number of files that can be sent within one message. <br />
`acceptedFormats` is used to define the accepted file formats. It is a singular string that uses the same syntax
as the [accept](https://developer.mozilla.org/en-US/docs/Web/HTML/Attributes/accept) attributes
in [input elements](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input/file). It is also worth noting that the [`dragAndDrop`](HERE) feature will also
use this to verify the dropped files. <br />
`infoModal` is used to display noteworthy information when the user clicks the button to search for a file. <br />

<ComponentContainer>
  <AiAssistantBrowser
    introMessage={`Use the image button or drop a file to attach it to the next outgoing message.`}
    service={{
      custom: {
        demo: true,
        images: {
          files: {
            maxNumberOfFiles: 2,
            acceptedFormats: '.jpg,.png',
          },
        },
      },
    }}
  ></AiAssistantBrowser>
</ComponentContainer>

<LineBreak></LineBreak>

### `InfoModal` {#InfoModal}

- Type: {`textMarkDown: string`, `openModalOnce?: boolean`, [`containerStyle?: CustomStyle`](./sharedTypes/#CustomStyle)}

- Default: _{openModalOnce: false}_

A modal that is triggered when a user clicks on a button to upload a file. It is useful for scenarios when a user needs some
clarification of how their data will be handled or they need to be aware of other noteworty information. <br />
`textMarkDown` is a string that uses [Markdown](https://www.markdownguide.org/basic-syntax/) syntax to generate the content of the modal.
Use this [Playground](https://jonschlinkert.github.io/remarkable/demo/) to customize your content accordingly. <br />
`openModalOnce` controls whether the modal is only opened up the first time the user clicks a button or every time. <br />
`containerStyle` is used to augment the basic styling of the modal element.

<ComponentContainer>
  <AiAssistantBrowser
    introMessage={`Click the image button to open up the modal.`}
    service={{
      custom: {
        demo: true,
        images: {
          files: {
            infoModal: {
              textMarkDown:
                'Please take note of the terms of our service: [link](https://www.youtube.com/watch?v=dQw4w9WgXcQ&ab_channel=RickAstley).',
              openModalOnce: true,
              containerStyle: {borderRadius: '5px'},
            },
          },
        },
      },
    }}
  ></AiAssistantBrowser>
</ComponentContainer>

### `MessageLimits` {#MessageLimits}

- Type: {`maxMessages?: number`, `totalMessagesMaxCharLength?: number`}

// WORK - display the intercepted messages

Used to limit the message content that is going to be sent to the target service (for both user and AI). <br />
`maxMessages` is the maximum number of messages that are going to be sent. <br />
`totalMessagesMaxCharLength` is the maximum number of text characters that are going to be sent and will inherently limit
the number of included messages. This is particularly useful if the target service has a token limit. <br />
If `maxMessages` number is hit before `totalMessagesMaxCharLength`, `maxMessages` will take presendence. <br />
These limits do not include the [`introMessage`](HERE).

<ComponentContainer>
  <AiAssistantBrowser
    introMessage={`Observe the data that is going to be sent below.`}
    service={{
      custom: {
        demo: true,
        totalMessagesMaxCharLength: 20,
        maxMessages: 2,
        requestInterceptor: (data) => {
          console.log(JSON.stringify(data));
          return data;
        },
      },
    }}
  ></AiAssistantBrowser>
</ComponentContainer>

### `RequestInterceptor` {#RequestInterceptor}

- Type: ([`details: RequestDetails`](#RequestDetails)) => [`RequestDetails`](#RequestDetails)

// WORK - display the result data below the component

A method that can be used to intercept outgoing HTTP requests and preprocess their data before they are sent. It MUST return a [`RequestDetails`](#RequestDetails) type object.

<ComponentContainer>
  <AiAssistantBrowser
    introMessage={`Send a message to intercept the request.`}
    service={{
      custom: {
        demo: true,
        requestInterceptor: (data) => {
          console.log(JSON.stringify(data));
          return data;
        },
      },
    }}
  ></AiAssistantBrowser>
</ComponentContainer>

<LineBreak></LineBreak>

#### `RequestDetails` {#RequestDetails}

- Type: {`body: any`, `headers?: {[string]: string}`}

Object containing data that is going to be used in the HTTP request to the target server. <br />
`body` can contain any form of data. <br />
`headers` is an optional object that contains the headers that are assigned to the request header. It is used as follows: <br /> _{ “Header name”: “Header value” }_

<LineBreak></LineBreak>

### `ResponseInterceptor` {#ResponseInterceptor}

- Type: (`response: any`) => [`Result`](#Result)

A method that can be used to intercept incoming HTTP responses and preprocess them before the component reads them.
It MUST return a [`Result`](#Result) type object. This is not supported for [`stream`](#stream) messages.

<ComponentContainer>
  <AiAssistantBrowser
    introMessage={`Send a message to intercept the response.`}
    service={{
      custom: {
        demo: true,
        responseInterceptor: (data) => {
          console.log(JSON.stringify(data));
          return data;
        },
      },
    }}
  ></AiAssistantBrowser>
</ComponentContainer>

<LineBreak></LineBreak>

### `Result` {#Result}

- Type: {`text?: string`, [`files?: MessageFile[]`](../messages#MessageFile)}

Object containing response data from the target server. It must contain either one of the following properties: <br />
`text` is a string that represents a textual message response. The component analyzes the string for [Markdown](https://www.markdownguide.org/basic-syntax/) properties
and renders the message accordingly. This is useful for displaying code snippets, paragraphs etc. Use this [Playground](https://jonschlinkert.github.io/remarkable/demo/) for reference. <br />
`files` is an array of objects that contain information about response files.

<LineBreak></LineBreak>

### `stream` {#stream}

- Type: `boolean`

If your service supports [`server-sent events`](https://developer.mozilla.org/en-US/docs/Web/API/Server-sent_events/Using_server-sent_events) via the `text/event-stream` MIME type, you
can stream your messages into the chat. (Please see [`examples`](https://github.com/OvidijusParsiunas/ai-assistant/tree/main/other-packages/servers) on how to set up a server).
This is only supported for text messages.

<ComponentContainer>
  <AiAssistantBrowser
    introMessage={`The response message bubble will be populated gradually with text events.`}
    service={{custom: {demo: true, stream: true}}}
  ></AiAssistantBrowser>
</ComponentContainer>

<LineBreak></LineBreak>

### `displayServiceErrorMessages` {#displayServiceErrorMessages}

- Type: `boolean`

This is used to display the returned error messages from the service in the error message bubble.

<ComponentContainer>
  <AiAssistantBrowser
    service={{custom: {displayServiceErrorMessages: true, demo: {displayErrors: {service: true}}}}}

</ComponentContainer>
