# Existing Services

Deep Chat can be configured to connect to an existing API service. The component's configuration
(such as file upload buttons / message limits) will automatically be changed to suit the target API.
This can be overwitten manually. <br />

:::caution
By default, the use of [`existingService`](#existingService) and [`APIKey`](#APIKey) will have the API key exposed on the browser. This is
intended for **prototyping purposes ONLY** and before deploying this component to an environment - add the [`request`](../service#request) property to
instead direct all requests through your own service to use the key there.
Check [examples](https://github.com/OvidijusParsiunas/deep-chat/tree/main/other-packages/servers).
:::

### `existingService` {#existingService}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `openAI?:` {[`OpenAI`](openAI#openAI), [`APIKey`](#APIKey)}, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `huggingFace?:` {[`HuggingFace`](huggingFace#huggingFace), [`APIKey`](#APIKey)}, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `azure?:` {[`Azure`](azure#azure), [`APIKey`](#APIKey)}, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `cohere?:` {[`Cohere`](cohere#cohere), [`APIKey`](#APIKey)}, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `assemblyAI?:` {[`AssemblyAI`](assemblyAI#assemblyAI), [`APIKey`](#APIKey)}, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `demo?:` [`Demo`](demo#demo) <br />
  }

## Types

Shared types for the `existingService` property.

### `APIKey` {#APIKey}

- Type: {`key?: string`, `validateKeyProperty?: boolean`}

These object properties are used to load up the chat view without the user having to insert the API key. <br />
`key` is the target service's API key required for authentication. <br />
`validateKeyProperty` is used to validate the value that is set for the `key` property. This will render a loading circle before the chat view
is loaded up during the validation period.

:::caution
This object is intended to be used for **prototyping purposes ONLY** and should not be deployed to an environment as users with malicious intent will be able to access your key in the browser.
Check [examples](https://github.com/OvidijusParsiunas/deep-chat/tree/main/other-packages/servers) on how to direct all requests through your own service.
:::
