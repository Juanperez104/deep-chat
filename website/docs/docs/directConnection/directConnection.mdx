# Direct Connection

Deep Chat can be configured to bypass the use of an intermediary service and connect to a popular AI service API directly from the UI. The component's configuration
(such as file upload buttons / message limits) will automatically be changed to suit the target API. This can nevertheless be overwitten manually. <br />

:::caution
The use of [`directConnection`](#directConnection) and [`APIKey`](#APIKey) properties will expose the API key on your browser. This is
intended for **prototyping / demo purposes ONLY**. Before deploying this component, you should instead use the [`request`](../connect#request) property to direct
all the requests to your own service. Read more about this in [`Connect`](../connect) and check [examples](https://github.com/OvidijusParsiunas/deep-chat/tree/main/other-packages/servers).
:::

### `directConnection` {#directConnection}

- Type: {<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `openAI?:` {[`OpenAI`](openAI#openAI), [`APIKey`](#APIKey)}, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `huggingFace?:` {[`HuggingFace`](huggingFace#huggingFace), [`APIKey`](#APIKey)}, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `azure?:` {[`Azure`](azure#azure), [`APIKey`](#APIKey)}, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `cohere?:` {[`Cohere`](cohere#cohere), [`APIKey`](#APIKey)}, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `assemblyAI?:` {[`AssemblyAI`](assemblyAI#assemblyAI), [`APIKey`](#APIKey)}, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `demo?:` [`Demo`](demo#demo) <br />
  }

## Types

Shared types for the `directConnection` property.

### `APIKey` {#APIKey}

- Type: {`key?: string`, `validateKeyProperty?: boolean`}

These object properties are used to load up the chat view without the user having to insert the API key. <br />
`key` is the target service's API key required for authentication. <br />
`validateKeyProperty` is used to validate the value that is set for the `key` property. This will render a loading circle before the chat view
is loaded up during the validation period.

:::caution
This object is intended to be used for **prototyping purposes ONLY** and should not be deployed to an environment as it can be accessed in the browser.
Before going live, use the [`request`](../connect#request) property in combination with your own service to safely access your key there.
Read more about this in [`Connect`](../connect) and check [examples](https://github.com/OvidijusParsiunas/deep-chat/tree/main/other-packages/servers).
:::
